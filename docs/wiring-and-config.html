Terminal Map — What Connects Where
5-Terminal Gauge (36Ω coil)
+ Coil positive → DRV8871 OUT1
− Coil negative → DRV8871 OUT2
res Coil tap (~770Ω from +) Not used
m.c. Moving contact (needle — live) To wire → GPIO32
f.c. Fixed contact (both stops wired together) To wire → GND
Sound Wave Relay (3kΩ coil)
Coil + Coil positive → GPIO26 (DAC2)
Coil − Coil negative → GND
Fixed contact Alarm switch (stationary) Not used
Moving contact Alarm switch (paddle) Not used
Drive Strategy

Milliamp Meter

GPIO25 (DAC1) → 820Ω → Meter + → Meter − → GND

Same as before. 4mA full scale, resistor limits current. Simple analog drive.

Sound Wave Relay (3kΩ)

GPIO26 (DAC2) → Coil + → Coil − → GND

This is a vintage polarized relay, not an analog meter — the paddle snaps between two positions rather than sweeping smoothly. Originally connected to a microphone in the vault to detect drilling vibrations. At 3kΩ, max current from 3.3V is only 1.1mA — no resistor needed. The relay snaps closed above a threshold current; use it as an on/off indicator.

5-Terminal Gauge (36Ω)

GPIO16/17 (PWM) → DRV8871 → + and − terminals

At 36Ω, direct drive would pull ~92mA — too much for the DAC. The DRV8871 handles the current. PWM duty cycle controls needle position.

Alarm Contacts — 5-Terminal (to wire)

To connect: m.c. → GPIO32 (INPUT_PULLUP) · f.c. → GND

Both side contacts are wired together at f.c. — the needle trips the same circuit whether it swings left or right (center-off design). ESPHome binary sensor "5-Terminal Alarm Contact" is already enabled in the config and ready — just run the two wires to activate it.

3kΩ Gauge — Why No Resistor?

At 3kΩ, the maximum current from 3.3V is only 1.1mA. The ESP32 DAC can source up to ~12mA, so there's huge margin. The coil's own resistance is the current limiter. If you want extra safety, add a 1kΩ resistor in series (drops max to ~0.8mA).

36Ω Gauge — Why DRV8871?

At 36Ω, even 3.3V would push ~92mA — far more than the ESP32 can source. The DRV8871 handles the current, and PWM duty cycle controls how much actually flows. The max_power setting in ESPHome caps the duty cycle as a safety limit.

Unused Terminals

The 5-terminal alarm contacts are now wired — m.c. → GPIO32, f.c. → GND. The 4-terminal relay contacts (moving/fixed contact) are still unused; wire to GPIO33 if you want that relay's snap to be readable in HA too.

VM Voltage

Start with 3.3V on the DRV8871's VM pin. If the 36Ω gauge doesn't deflect enough, move VM to the ESP32's VIN/5V pin for more headroom. Reduce max_power when you increase voltage.

      - output.set_level: { id: gauge5_in1, level: 0.0 }
      - output.set_level: { id: gauge5_in2, level: 0.0 }
      - delay: 500ms
      # Sweep up / right
      - output.set_level: { id: milli_dac, level: 1.0 }
      - output.set_level: { id: gauge4_dac, level: 1.0 }
      - output.set_level: { id: gauge5_in1, level: 1.0 }
      - delay: 600ms
      # Sweep down / left (5-terminal only)
      - output.set_level: { id: gauge5_in1, level: 0.0 }
      - output.set_level: { id: gauge5_in2, level: 1.0 }
      - delay: 600ms
      # Return to rest
      - output.set_level: { id: milli_dac, level: 0.0 }
      - output.set_level: { id: gauge4_dac, level: 0.0 }
      - output.set_level: { id: gauge5_in2, level: 0.0 }

What you get in Home Assistant: Three slider entities — "Milliamp Gauge" (0–100%), "4-Terminal Gauge" (0–100%), and "5-Terminal Gauge" (-100 to +100) for manual control, plus live solar sensor mappings.

Solar integration (SolarEdge 16.5kW): The milliamp meter tracks current solar generation — sensor.solaredge_current_power scaled to 16,500W full scale. The 5-terminal gauge currently shows generation vs 50% capacity (needle centers at 8,250W). When a grid energy meter becomes available, swap the 5-terminal entity for net grid power to show import/export instead.

Sound wave relay (4-terminal): This is a vintage polarized relay, not an analog meter. The paddle snaps between positions rather than sweeping. Best used as an on/off indicator — e.g., snap when solar is actively exporting, or when a security event triggers. The manual slider still works as a test control.

Alarm contacts — config ready, wiring pending: The ESPHome binary sensor "5-Terminal Alarm Contact" is enabled and waiting. Both side contacts are wired together at f.c. (center-off design) — needle trips the same circuit in either direction. To activate: run m.c. → GPIO32 and f.c. → GND. Will appear in HA automatically once the wires are connected.

Safety: The 5-terminal gauge outputs start at max_power: 0.10 (10% duty cycle cap). The two DAC gauges are inherently current-limited by their coil resistance and the 820Ω resistor. Increase max_power gradually during calibration.

Calibration — Step by Step
1
Flash the config as-is with max_power: 0.10 on the DRV8871 outputs. The DAC-driven gauges are self-limiting so they're safe from the start.
2
Test the milliamp meter first — it's the simplest. Move the "Milliamp Gauge" slider to 50% in HA. The needle should move to roughly mid-scale. At 100% it should be near the "4" mark. If it goes past 4, reduce max_power on milli_dac until 100% = full scale.
3
Test the sound wave relay (4-terminal, 3kΩ). This is a polarized relay, not an analog meter — the paddle snaps between positions rather than sweeping smoothly. Move its slider from 0% to 100% slowly and listen/watch for a snap. Note the threshold where it trips. It won't give proportional deflection; just on/off behavior above the pull-in current. That's normal and correct.
4
Test the 5-terminal gauge (36Ω) carefully. Move its slider to +20, then +50. Watch the needle. If it barely moves at +50, increase max_power to 0.20 and re-flash. If it slams to the stop at +20, drop to 0.05. Repeat until +100 gives full-scale deflection without hitting the mechanical stop.
5
Test the reverse direction. Move the 5-terminal slider to -20, -50, -100. Verify it deflects the opposite way. If one direction is weaker, add an asymmetry correction in the lambda (multiply the weak side by a factor like 1.15).
6
If 3.3V isn't enough for the DRV8871: Move the VM wire from 3V3 to the ESP32's VIN/5V pin. Drop max_power back down to 0.05 and recalibrate — you'll have ~50% more voltage headroom.
7
Lock in your final max_power values. These act as permanent safety limits. Even a buggy automation can't overdrive the gauges past these caps.

Protect the Vintage Coils

These are 80+ year old instruments with irreplaceable fine wire coils. Always err on the side of too little current rather than too much. You can always increase — you can't un-burn a coil.

Needle Damping

If the needle oscillates or bounces when values change, add an exponential_moving_average filter to the sensor, or use small delay steps in the lambda to ramp the output gradually instead of jumping directly to the target.

Sticky Movements

If a needle doesn't move at all, it may be stuck from age. Gently tap the gauge face while applying a signal. If it breaks free and moves smoothly, the pivot just needed unseating. A tiny drop of clock oil on the jewel bearings can help long-term.

The 4-Terminal Might Be Bidirectional

Even though it only has Coil + and Coil −, some galvanometers can deflect both directions based on current polarity. Try swapping the DAC wire between Coil + and Coil − to see if the needle moves the other way. If it does, you could use the DRV8871 approach for this one too (or just pick the direction you want).
